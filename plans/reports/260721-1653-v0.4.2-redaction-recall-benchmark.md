## Executive summary

This release adds an executable synthetic benchmark for the result-redaction backstop and hardens context-aware detection without expanding the shell parser. Public redaction function signatures remain unchanged.

The detector now covers credential-bearing HTTP/FTP URLs, Basic and other Authorization schemes, query parameters, camelCase and hyphenated keys, quoted values with whitespace or escaped quotes, and approved plural structured fields. Key matching was tightened so configuration and language-metadata fields such as `passwordPolicy`, `semanticTokenType`, and `colorTokens` remain unchanged.

## Baseline evidence

The exploratory corpus detected 26 of 33 contextual cases (78.8%), with zero false positives across 12 benign text cases. It missed query tokens, hyphenated headers, camelCase secret fields, passphrases, private-key fields, HTTP user-info credentials, and Basic authorization. Structured redaction also changed an opaque `id` under a non-sensitive key.

After the initial corpus expansion, the pre-fix executable gate detected 31 of 42 contextual cases. Independent review then added quoted values, general Authorization schemes, plural sensitive collections, pure-alphabetic contextual values, and metadata false-positive controls.

## Current benchmark contract

- Contextual/known-format recall must be 100%.
- Committed benign text controls must have zero false positives.
- Structured sensitive and benign totals are pinned explicitly; a reduced or empty corpus cannot pass silently.
- Unlabeled entropy remains observational and does not affect the release result.
- Large-output timing is recorded for local comparison but is not a cross-machine performance claim.

## Security boundary and residual risk

- Pattern-based redaction cannot identify every arbitrary secret representation.
- Opaque high-entropy values without a credential-bearing context remain visible by design because they are indistinguishable from many hashes and identifiers.
- Runtime-generated interpreter access remains outside static shell-path analysis.
- This output backstop is not an operating-system sandbox. Untrusted workloads still require process, filesystem, network, and credential isolation.
